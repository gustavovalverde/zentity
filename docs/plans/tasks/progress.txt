## Phase 1: Write Convergence (PRD-17)

### Completed: Schema + Write Path Migration

Schema changes:
- zk_proofs → proof_artifacts: added proofSystem discriminator, metadata JSON, removed Noir-specific columns (isOver18, circuitType, noirVersion, circuitHash, bbVersion, verificationKeyHash, verificationKeyPoseidonHash)
- zk_proof_sessions → proof_sessions
- New: verification_checks table (7 check types, unique on verificationId+checkType)
- zkChallenges FK updated to reference proof_sessions

OCR write path:
- storeProofProcedure writes proofSystem: "noir_ultrahonk", Noir fields in metadata JSON
- Calls materializeVerificationChecks after proof storage

NFC write path:
- submitResult stores each ZKPassport ProofResult in proof_artifacts (proofSystem: "zkpassport")
- Computes proofSetHash and upserts attestation_evidence (was missing for NFC)
- Calls materializeVerificationChecks after writes

Materialization engine:
- materializeVerificationChecks(userId, verificationId) in identity/verification/materialize.ts
- Fetches proofs + claims, runs deriveComplianceStatus, maps 7 boolean checks to verification_checks rows
- OCR evidence: zk_proof, signed_claim, dedup_key
- NFC evidence: chip_claim, commitment, nullifier
- Idempotent via upsert on unique constraint

Query layer updates:
- All query functions renamed (createProofSession, getProofSessionById, etc.)
- getUserAgeProof/Full reads isOver18 and Noir metadata from JSON
- getAllVerifiedProofsFull returns proofSystem field

Files changed:
- src/lib/db/schema/crypto.ts — schema rename + new table
- src/lib/db/queries/crypto.ts — query functions
- src/lib/db/queries/identity.ts — table references
- src/lib/db/queries/auth.ts — comments
- src/lib/trpc/routers/zk/proof.ts — OCR write path
- src/lib/trpc/routers/zk/challenge.ts — session imports
- src/lib/trpc/routers/passport-chip.ts — NFC write path
- src/lib/identity/verification/materialize.ts — NEW
- src/lib/assurance/data.ts — import rename
- src/app/(dashboard)/dashboard/_components/identity-card.tsx — import rename
- src/app/api/zk/nationality-proof/verify/route.ts — import rename
- src/test/db-test-utils.ts — table reference + verificationChecks delete
- e2e/global-setup.ts — seed SQL
- 4 test files updated for new function signatures

Decisions:
- proofPayload kept as text (not blob): both Noir hex and ZKPassport proof strings are text-encodable
- ComplianceInput.zkProofs property NOT renamed (deferred to Phase 2 unified model)
- AssuranceState.zkProofsComplete NOT renamed (deferred to Phase 2)
- materializeVerificationChecks is awaited (not fire-and-forget) — 7 upserts are fast

Blockers: None

## Phase 2: Read Convergence + Cleanup (PRD-17)

### Completed: Unified Read Model + Consumer Migration + Dead Code Removal

Unified read model:
- getUnifiedVerificationModel(userId) in identity/verification/unified-model.ts
- Reads verification_checks (7 materialized checks), proof_artifacts (summaries), identity_verifications, identity_bundles, encrypted_attributes, encrypted_secrets, blockchain_attestations
- Derives compliance level from materialized checks (no re-running deriveComplianceStatus)
- React.cache() wrapped for per-request dedup in server components
- isFheComplete inlined to avoid circular dep with assurance module

Assurance layer migration:
- gatherVerificationData replaced with getUnifiedVerificationModel + toAssuranceInput conversion
- toAssuranceInput preserves assurance semantics: documentVerified from verifiedAt, zkProofsComplete from proof summaries, liveness/faceMatch from compliance checks
- getAssuranceState and getAssuranceForOAuth both use unified model
- hasPasskeyCredentials login method fallback preserved

OAuth claims migration:
- buildProofClaims reads from unified model (single query instead of 3 parallel)
- buildOidcVerifiedClaims reads from unified model
- mapVerificationClaims → mapComplianceToClaims (takes compliance struct, not ComplianceResult)
- Output shape unchanged: 15 OIDC claim keys preserved

Attestation migration:
- attestation.submit reads birthYearOffset, numericLevel, issuerCountry from unified model
- Single getUnifiedVerificationModel call replaces getVerificationStatus + getSelectedVerification

New tRPC endpoints:
- zk.getChecks — returns method, level, verified, 7 checks with source/evidenceRef
- zk.getProofs — returns method, proof summaries (system, type, hash, verified, createdAt)
- Both backed by getUnifiedVerificationModel

MCP my_proofs rewrite:
- Calls zk.getChecks + zk.getProofs (replaces zk.getUserProof + zk.getAllProofs)
- Returns verificationMethod, verificationLevel, verified, checks with human-readable source labels
- NFC chip users now see non-empty results with "passport chip" source labels

Dead code removed:
- getUserAgeProof, getUserAgeProofFull, getAllVerifiedProofsFull from crypto.ts
- getUserProofProcedure, getAllProofsProcedure from zk/proof.ts
- parseProofMetadata, getCiphertextInfo private helpers from crypto.ts
- AgeProofFull/AgeProofSummary imports from crypto.ts
- gatherVerificationData and all private helpers from assurance/data.ts

Client-side updates:
- zk/client.ts: getUserProof/getAllProofs → getChecks/getProofs
- Inspector page rewritten to show verification checks + proof summaries (no raw proof data)

Tests updated:
- data.integration.test.ts: 4 tests now call materializeVerificationChecks before asserting
- crypto.integration.test.ts: removed 2 tests for deleted functions
- claims.test.ts: mocks getUnifiedVerificationModel instead of getVerificationStatus
- attestation.test.ts: added getUnifiedVerificationModel mock

Files changed:
- src/lib/identity/verification/unified-model.ts — NEW
- src/lib/assurance/data.ts — rewritten
- src/lib/auth/oidc/claims.ts — rewritten
- src/lib/trpc/routers/attestation.ts — migrated
- src/lib/trpc/routers/zk/proof.ts — new procedures, removed old
- src/lib/trpc/routers/zk/router.ts — new endpoints, removed old

- src/lib/privacy/zk/client.ts — new client functions
- src/app/(dashboard)/dashboard/developer/inspector/page.tsx — rewritten
- apps/mcp/src/tools/my-proofs.ts — rewritten
- 4 test files updated

Decisions:
