## Phase 3: Release Handle Scoping and Atomic Redemption (CIBA Notifications Remediation)
- PRD ref: Findings 2, 4 (release handle scope collision, non-atomic release)
- Ephemeral release handle store re-keyed by authReqId with userId reverse index
- customAccessTokenClaims guarded: only consumes handles when identity scopes present
- Non-CIBA grants (auth-code, refresh, token-exchange) can no longer steal CIBA handles
- Release endpoint uses 3-phase atomic claim: approved → claiming → redeemed
- Rollback to "approved" if decrypt/sign fails mid-request
- Concurrent callers get exactly one success (SQLite serialized writes)

## Phase 1: Push Notification Safety (CIBA Notifications Remediation)
- PRD ref: Findings 1, 3 (push ownership transfer, service worker bypass)
- Subscribe endpoint upserts userId on conflict — transfers ownership when different user resubscribes same browser endpoint
- Push payload includes requiresVaultUnlock flag derived from identity scopes in CIBA request
- Service worker suppresses quick-approve action for identity-scoped requests; deny stays
- Identity-scoped notification clicks always open approval page (never direct API call)
- Non-identity CIBA requests retain both approve/deny quick actions
- 1 new integration test: endpoint ownership transfer across two users
- 1 new unit test: requiresVaultUnlock flag preserved in serialized push payload

### Files changed
- apps/web/src/app/api/push/subscribe/route.ts (add userId to upsert set)
- apps/web/src/lib/auth/auth.ts (add requiresVaultUnlock to push payload)
- apps/web/public/push-sw.js (conditional actions + identity-scoped click routing)
- apps/web/src/app/api/push/__tests__/push-routes.integration.test.ts (ownership transfer test)

## Phase 1: Protected Resource Metadata + Discovery (MCP Compatibility)
- PRD ref: Phase 1 (User stories 1, 9, 12)
- RFC 9728 endpoint at /.well-known/oauth-protected-resource
- Discovery metadata enriched with client_id_metadata_document_supported + resource_indicators_supported
- 7 new tests (5 RFC 9728 route, 2 MCP discovery flags) + 1 pre-existing CIBA test fix (ping mode)
- All acceptance criteria met

# Key decisions made and reasoning (architectural decisions)
- Route is a simple Next.js handler (no [[...issuer]] catch-all) — RFC 9728 defines single path
- Scopes built inline importing PROOF_SCOPES/IDENTITY_SCOPES constants — stays in sync without coupling to auth.ts
- resource_signing_alg_values_supported: ["EdDSA"] — matches access token signing alg
- Cache-Control: public, max-age=3600 (1hr) — per PRD spec, longer than discovery's 15s since this changes rarely
