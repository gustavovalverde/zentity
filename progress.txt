## Phase 1: Step-Up Hook Session Fix — COMPLETE

### Task
Fix step-up auth hook so acr_values/max_age enforcement actually runs on /oauth2/authorize.

### Bug
`resolveSessionToken()` extracted full signed cookie (token.hmac-sig) but DB stores raw token → query never matched → hook always short-circuited.

### Fix
- Replaced manual cookie parsing (`SESSION_COOKIE_RE`, `resolveSessionToken`, `resolveSession`, `sessions` import) with `getSessionFromCtx(ctx)` from `better-auth/api`
- Maps `{ session.createdAt, user.id }` → `SessionInfo` at call sites
- Removed unused `db` param from `enforceFromQuery` (only needed in `enforceFromPar` for PAR records)
- Updated integration tests to sign session cookies via `makeSignature` from `better-auth/crypto`

### Files changed
- `apps/web/src/lib/auth/oidc/step-up-hook.ts` — removed broken session resolution, use getSessionFromCtx
- `apps/web/src/lib/auth/__tests__/step-up-auth.integration.test.ts` — sign session cookies with HMAC
