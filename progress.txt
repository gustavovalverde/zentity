  - EDIT: src/app/api/auth/oauth2/jwks/route.ts — filter ECDH-ES keys from public JWKS
- Arch decisions:
  - KEK derived via SHA-256 to normalize any input to 32 bytes
  - Envelope format: {"v":1,"iv":"...","ct":"..."} — version field enables future format changes
  - Plaintext detection: if stored value doesn't start with {"v":, treat as plaintext (migration compat)
  - No KEK set = no-op (keys stored/read as plaintext, for dev convenience)
  - disablePrivateKeyEncryption: true KEPT — removing it would make better-auth encrypt with BETTER_AUTH_SECRET in its own format, breaking our signers that query the same jwks table
  - JARM ECDH-ES filtered from /oauth2/jwks (decryption keys don't belong in verification JWKS)
