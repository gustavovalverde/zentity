# Architectural decisions
- Concurrent release test accepts 400 OR 410 for loser — Node.js single-threaded async means the second caller may re-read DB after status="claiming", hitting status guard (400) instead of atomic claim race (410). Both prevent double PII delivery.

# Architectural decisions
- Vendor sequencing: broke Promise.all parallelism in createUserTokens — access token created first (await), then refresh token + ID token in parallel. Enables at_hash in customIdTokenClaims without post-hoc re-signing.
- Signing alg resolution: reads metadata.id_token_signed_response_alg from client metadata (same source of truth as jwt-signer.ts), defaults to RS256.
