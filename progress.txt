## Task 02: Compliance Schema Cleanup & Server-Derived birthYearOffset
Status: Complete

### Changes
1. Removed `livenessPassed` and `faceMatchPassed` columns from identityVerificationDrafts
   - All read sites now derive from scores via policy thresholds (ANTISPOOF_REAL_THRESHOLD, ANTISPOOF_LIVE_THRESHOLD, FACE_MATCH_MIN_CONFIDENCE)
   - Write sites (upsertIdentityDraft, handler.ts, liveness.ts) no longer set these
   - Single source of truth: raw scores, not redundant booleans
2. Server-derived `birthYearOffset` in attestation.submit
   - Added `birthYearOffset` column to identity_verifications
   - OCR path: computed from dobDays during prepareDocument via new dobDaysToBirthYearOffset()
   - NFC path: computed from birthdate during submitResult
   - attestation.submit reads from verification record; client no longer supplies it
   - Client component (on-chain-attestation.tsx) no longer does vault unlock for birth year
