# Decisions made and why
- Tests use auth.handler() directly (same as toNextJsHandler) with response envelope unwrapping
- JWT act/authorization_details tests set resource=authIssuer because access tokens are JWT only when audience is present (opaque otherwise)
- Integration test (not unit) because it exercises the full plugin dispatch chain
- Rebuilt zentity-combined with merges (not cherry-picks) to preserve Git history and prevent future phantom conflicts
- TokenResponse type and createUserTokens exported from oauth-provider so CIBA plugin can import them
- Custom grant handler type uses Promise<TokenResponse> (matching oidc-haip refactor) instead of Promise<Response>
