- 10 integration tests covering full CIBA token flow: pending, approved, rejected, expired, slow_down, replay prevention, client mismatch, act claim, authorization_details forwarding

# Decisions made and why
- Tests use auth.handler() directly (same as toNextJsHandler) with response envelope unwrapping
- JWT act/authorization_details tests set resource=authIssuer because access tokens are JWT only when audience is present (opaque otherwise)
- Integration test (not unit) because it exercises the full plugin dispatch chain: Zod parsing → customGrantTypeHandlers → CIBA grant handler → createUserTokens
