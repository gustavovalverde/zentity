# Tasks completed in this session

## Phase 1.1: Enable CIBA ping mode server-side (Plan §Phase 1, AC 1-5)
- Enabled `deliveryModes: ["poll", "ping"]` in ciba() plugin config
- Implemented `resolveClientNotificationEndpoint` — reads `backchannel_client_notification_endpoint` from client `metadata` JSON column
- Updated discovery metadata to advertise `["poll", "ping"]`
- 8 integration tests: endpoint resolution, fallback-to-poll, request-body override, ping-mode polling, authorization_pending, request creation, poll regression, missing-token fallback

# Key decisions made and reasoning (architectural decisions)
- `resolveClientNotificationEndpoint` queries `oauthClients.metadata` directly via Drizzle (already imported in auth.ts) rather than going through the better-auth adapter — simpler, no indirection
- Ping mode stores notification endpoint/token per CIBA request (not per client) — allows per-request endpoint override via `client_notification_uri` in bc-authorize body
